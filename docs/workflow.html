Sevilla FC Retail Analytics

5-Step Validation Workflow — click any step for details

Step 1: Classify Question

gpt-4o-mini Input: question

Determines if the question is about SFC retail, SFC non-retail, or general knowledge.

ClassificationAction
sfc_retailContinue to Step 2
sfc_non_retailAnswer directly with warning (exit)
generalAnswer directly with warning (exit)

Step 2: Validate Retail Domain

gpt-4o-mini Input: question

Decomposes into 3 dimensions and validates each:

  • Concept: Is this a product SFC sells? Is the metric measurable?
  • Date range: Between 2012 and today?
  • Location: Within SFC's network? On Earth?

Also rejects: privacy-violating queries (individual customers/employees), non-EUR currency, non-measurable aggregations.

Step 3: Map to Ontology

gpt-4o-mini (disambiguation only) Input: concept + question

Maps product concepts to the 3-level hierarchy (10 families → 21 subfamilies → 3,284 product types).

StageMethod
TranslationStatic EN→ES dictionary (~60 terms)
Exact matchName or path part equality
SubstringTerm found in search text
FuzzySequenceMatcher ≥ 0.6
DisambiguationLLM picks correct path when duplicates exist

Step 4: TLK Lookup → SQL

o4-mini (reasoning) Input: question + ontology context + date range

Finds or generates the SQL query with mechanical confidence assignment:

ConfidenceConditionMethod
🟢 ACCURATESimilarity ≥ 0.6Fill TLK template placeholders
🟡 MEDIUMSimilarity 0.4–0.6LLM adapts closest template
🔴 LOWNo matchLLM generates from DDL + ontology

Step 5: Execute & Summarize

gpt-4o-mini (summary) Input: SQL + confidence + question

Executes SQL via AWS Athena, returns CSV results, and generates natural language summary.

  • Results returned as CSV with row count
  • LLM summarizes findings in 2-3 sentences
  • LOW confidence queries include a warning banner
  • Monetary values reported in EUR
